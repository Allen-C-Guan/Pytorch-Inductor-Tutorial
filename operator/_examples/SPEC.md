# 示例补全规范（Tier D 算子详解）

本规范用于给 `docs/docs/aten/` 下**每个 Tier D 详解条目**补一段极简「输入→输出示例」。已完成的范例章：`01-elementwise-arithmetic.md`（见 `div`/`pow`/`fmod`/`clamp` 四个示例，**严格照搬其风格**）。

## 1. 范围
- 每个 `### \`aten.sig\` — ...` 开头的 Tier D 条目，补**恰好一个** `**示例**` 段。
- 一个条目即使覆盖两个算子（如 `fmod 与 remainder`）也只补一个合并示例。
- **Tier C 速查表（章首表）不动；条目的其它五段文字不动**（除第 5 条的情形）。

## 2. 放置位置
在 `**作用与语义**` 段**之后**、自适应段（`**为什么需要这个算子…` 或 `**数值与精度**`）**之前**插入。只插入新段，不动周围。

## 3. 默认格式（绝大多数算子 —— REPL 转录）

```
**示例**　<一句话：这个例子展示什么>：

```python
>>> <输入定义与调用，带 # 行内注释点出关键>
<真实输出，逐字粘贴>
```
```

要点：
- 用高层 `torch.<op>(...)` 调用（不是 `aten.` 裸调用）。
- 张量尽量 2×3 / 3×4，**优先小整数**（比浮点直观）。
- ` ```python ` 围栏 + `>>>` 提示符；`**示例**` 后接全角空格「　」再一句说明 + 中文冒号「：」，空一行再代码块。
- 输出值**必须来自实跑**（第 5 条），绝不手写、绝不臆测。

## 4. 难点结构算子（仅指定清单，加「标注图」）
对这些算子，在 REPL 之后**再追加一段无标签围栏**的「输入张量 → 输出张量」标注，把 index→地址 / 坐标→源坐标 的映射画出来。标注里的值**必须与 REPL 实跑一致**。范例（gather）：

```
**示例**　<…>

```python
>>> a   = torch.tensor([[10, 11, 12], [20, 21, 22]])
>>> idx = torch.tensor([[2, 0], [1, 1]])
>>> torch.gather(a, 1, idx)
tensor([[12, 10],
        [21, 21]])
```

```
a   = [[10, 11, 12]    idx = [[2, 0]      out = [[12, 10]   # a[0,2], a[0,0]
       [20, 21, 22]]           [1, 1]]            [21, 21]]   # a[1,1], a[1,1]
```
```

是否加图、加在哪几个算子，由派给你的章节任务里**明确指定**；未指定的算子一律只用默认 REPL。

2. **在仓库根目录 `/models/torch-inductor/Allen` 下**运行：`source env.sh new && python docs/docs/aten/_examples/verify_chNN.py`（`env.sh` 在仓库根，必须在根目录 source）。
3. 把**实跑打印的真实输出**逐字粘进文档。**绝不手写**数值。
4. **随机类算子**（`rand`/`randn`/`randperm`/`dropout` 等）：脚本与示例开头都加 `torch.manual_seed(0)`，使数值可复现；正文加一句"已设种子，实际为随机"。
